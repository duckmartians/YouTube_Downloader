# YouTube 下载器 Pro

**[English](README.md) · [Tiếng Việt](README_vi.md) · [中文](README_zh.md) · [Español](README_es.md) · [العربية](README_ar.md) · [Русский](README_ru.md)**

一款简洁好用的桌面应用,用于下载 YouTube 视频、播放列表、频道和 Shorts —— 支持音频提取、缩略图、字幕及多视频并行下载。已内置 `ffmpeg` 与 JavaScript 运行时,**无需安装任何额外软件**。

---

## 主要功能

- 下载单个视频、整个播放列表、整个频道,或频道中的 Shorts 集合。
- 保存为 **MP4 视频**(最佳 / 1080p / 720p / 480p)或 **MP3 音频**(192 kbps)。
- 可选同时保存 **缩略图(JPG)** 和 **字幕(SRT)**。
- 最多 **同时下载 10 个视频**。
- **预览播放列表**,精确选择要加入队列的视频。
- 一键 **重试失败视频**。
- 将 **下载队列** 或 **仅失败视频** 导出为 CSV。
- 支持 **Cookie**(从浏览器扩展粘贴或选择文件)。
- 支持 **代理**(HTTP / HTTPS / SOCKS4 / SOCKS5,可附带认证)。
- 自动跳过已下载过的视频。
- 支持 **6 种语言**:English、Tiếng Việt、中文、Español、العربية、Русский。
- **`yt-dlp` 每 24 小时自动更新**,跟上 YouTube 的变化。

---

## 快速开始

1. 运行 `YouTubeDownloader.exe`。
2. 在 **链接** 框粘贴 YouTube 链接,按 **Enter**(或点击 **添加**)。
3. (可选)在 **选项** 卡片中调整画质、格式或附加项。
4. 点击 **开始下载**。

完成!文件会保存到 **设置** 卡片中显示的 **保存目录**(默认是程序旁边的 `youtube_downloads/` 文件夹)。

## 详细使用指南

### 1. 添加视频到队列

四种添加方式:

- 粘贴 **单个视频链接**(`https://www.youtube.com/watch?v=...`),按 **Enter** 或点击 **添加**。
- 粘贴 **播放列表链接**(`...?list=...`)—— 应用自动展开为单个视频。
- 粘贴 **频道链接**(`youtube.com/@channelname`)—— 应用获取该频道的常规视频 **和** Shorts 并缓存。
- 点击 **预览**(🔍)打开对话框列出播放列表/频道中的所有视频,勾选需要的视频后点击 OK。

添加后,视频会出现在右侧的 **队列** 表格中,状态为 `⏳ 等待中`。

> **提示 — 播放列表与频道会被缓存。** 当您粘贴一个频道链接时,应用会下载视频列表并保存到本地。之后您可以更改 **数量** 或在 **常规 / Shorts** 之间切换,队列会立即更新而无需重新获取。

### 2. 选择要下载的内容(选项卡片)

| 切换按钮 | 功能 |
|---|---|
| **下载媒体**(默认开启) | 保存为 MP4 视频 |
| **仅音频** | 仅保存音频为 MP3,192 kbps |
| **缩略图** | 保存封面为 JPG |
| **字幕** | 保存字幕为 SRT(自动选择最合适的语言) |

**下载媒体** 与 **仅音频** 互斥 —— 开启一个会自动关闭另一个。

如果两个都关闭,应用会跳过主下载,但若启用了缩略图/字幕仍会保存。适合只想要封面图或字幕的情况。

| 其他选项 | 范围 / 取值 | 说明 |
|---|---|---|
| **画质** | 最佳、1080p、720p、480p | 在音频模式下被禁用 |
| **数量** | 0–9999 | `0` = 不限制。仅适用于播放列表/频道。 |
| **视频类型** | 常规视频 / Shorts | 仅适用于播放列表/频道。 |
| **并发下载** | 1–10 | 同时下载的视频数 |

### 3. 开始与监控

点击 **开始下载**。状态栏显示总进度;队列中每一行都有自己的进度条。随时可点击 **停止** —— 会弹出确认对话框,然后所有正在进行的下载会被取消。

全部完成后,状态会切换为 **全部完成**。

### 4. 重试失败视频

如果某些行显示 `❌ 错误`,点击 **重试失败**。队列会重建为仅包含失败的行,并自动开始下载。已成功的视频不会被重试,避免重复下载。

### 5. 导出队列或错误列表

- **导出**:将整个队列(标题、URL、状态)保存为 `.csv` 文件。
- **导出错误**:仅保存失败的行。可用于分享问题列表,或保存以便稍后重试。

两个文件都使用 UTF-8 BOM,可正确在 Excel 中打开。

### 6. 添加前预览播放列表/频道

1. 在 **链接** 框粘贴播放列表或频道链接。
2. 点击 **预览**。新窗口会列出所有视频。
3. 勾选您要的视频。
4. 点击 **OK**。它们会被加入队列。

## Cookie(用于年龄限制或需要登录的视频)

点击顶部栏的 **Cookie** 打开 Cookie 对话框。

提供 cookie 有两种方式:

**A — 从浏览器扩展粘贴**(推荐)
1. 在 Chrome/Edge 安装 [Cookie-Exporter](https://chromewebstore.google.com/detail/cookie-exporter/fhnmmidekmgocpjdceeffppcodigillk) 扩展。
2. 在已登录状态访问 youtube.com,点击扩展,复制 JSON。
3. 粘贴到 **内容** 框,勾选 **启用**,点击 **保存**。

**B — 选择文件**
1. 点击 **浏览**,选择 `cookies.txt`(Netscape 格式)或 `.json` 文件。
2. 勾选 **启用**,点击 **保存**。

应用会自动识别 JSON 并在内部转换为 Netscape 格式。

> **何时真正需要 cookie:** 年龄限制视频、会员专享内容、出现 "Sign in to confirm you're not a bot" 错误时,或下载您自己的私人播放列表。

## 代理(可选)

点击顶部栏的 **代理** 打开代理对话框。

| 字段 | 说明 |
|---|---|
| **协议** | `http`、`https`、`socks4` 或 `socks5` |
| **IP / Host** | 例如 `127.0.0.1` 或 `myproxy.example.com` |
| **端口** | 必填 |
| **用户名 / 密码** | 可选,仅当代理需要认证时填写 |

点击 **检查代理** 在保存前验证 —— 应用会通过代理尝试访问 Google 并显示 ✅ 或 ❌。

## 文件保存位置

### 下载的媒体
默认情况下,文件保存到 `YouTubeDownloader.exe` 旁的 `youtube_downloads/` 文件夹。可在设置卡片中点 **浏览** 修改。

文件名使用以下模板:

- **视频:** `Uploader - Title [VideoID] [720p].mp4`
- **音频:** `Uploader - Title [VideoID].mp3`
- **缩略图 / 字幕:** 同前缀,扩展名为 `.jpg` 或 `.<lang>.srt`

文件名中带视频 ID,意味着 **两个同名视频不会互相覆盖**。

### 应用数据与配置
应用将内部数据保存在:

```
%APPDATA%\YouTubeDownloaderStandalone\
├── settings.json          ← 语言、代理、cookie 内容、上次 yt-dlp 检查时间
├── cookies.txt            ← Netscape 格式的 cookie
├── download_history.txt   ← 当前会话中已下载视频的 ID(应用启动时自动清除)
├── yt-dlp.exe             ← 自动下载的引擎
├── temp\                  ← 下载过程中的临时文件
└── temp_cache\            ← 已展开的播放列表/频道缓存
```

> **避免同一会话内重复下载:** `download_history.txt` 文件会记住自应用启动以来下载过的所有视频。如果您在同一会话中不小心两次添加同一视频,第二次会被跳过。该文件**每次启动应用时都会自动删除**,所以只需关闭并重新打开应用就能获得干净的状态 —— 视频可以自由地重新下载。

## 手动更新 yt-dlp

应用每次启动时,每 24 小时检查一次新版 `yt-dlp`。如果怀疑引擎已过时(例如 YouTube 刚更新),点击顶部栏的 **更新引擎** 立即强制刷新。

## 阅读日志

每行带有时间戳 `[HH:MM:SS]`。日志已过滤掉 yt-dlp 的常规进度信息,仅保留有用信息:

- `--- 开始处理视频: ...` —— 一个 worker 开始处理某个视频。
- `✅ 下载成功: ...` —— 完成。
- `[错误] ...` —— 出现错误;消息中含原因。
- `[任务 X] attempt 1/3 failed: ...` —— yt-dlp 报错;应用会以递增延迟(5 秒、10 秒、15 秒)重试最多 3 次。

## 故障排查

**"Sign in to confirm you're not a bot" / "Video unavailable"**
→ 从已登录的浏览器添加 cookie(参考上面的 Cookie 部分)。

**HTTP 429 / Too Many Requests**
→ YouTube 正在限制您的 IP 速率。等待 10–30 分钟,或使用代理。

**"Requested format is not available"**
→ 点击 **更新引擎** 刷新 `yt-dlp`。如果仍然出错,尝试不同的 **画质** 设置。

**日志显示 "⏭️ 已跳过:视频已在 download_history.txt 中"**
→ 在当前会话中您已经下载过此视频,所以被跳过了。关闭并重新打开应用 —— 历史记录会自动重置,下载将再次运行。

**点击开始后没有反应**
→ 确保队列不为空且 **保存目录** 路径有效(用 **浏览** 按钮以确保正确)。

**卡在"正在检查并更新下载工具…"**
→ 您的网络阻止了 `github.com`,或 `%APPDATA%\YouTubeDownloaderStandalone\` 是只读的。尝试以管理员权限运行一次以设置权限。
